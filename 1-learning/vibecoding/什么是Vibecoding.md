# 📖 什么是 Vibecoding

## 📌 一句话总结
Vibecoding 是一种用自然语言和 AI 协作编程的新方式——你描述想要什么，AI 帮你写代码。

## 🔍 核心理解

Vibecoding 不是「让 AI 替你写代码」，而是：
- **你是导演**：你决定做什么，把控方向和质量
- **AI 是副驾驶**：帮你实现细节、查找文档、调试错误
- **核心技能变了**：从「记住语法」变成「描述需求 + 审查代码 + 迭代优化」

### Vibecoding 的核心能力

1. **需求描述能力** — 把模糊想法变成清晰的指令
2. **代码审查能力** — 能看懂 AI 写的代码对不对
3. **调试思维** — 出了问题知道怎么排查
4. **架构判断力** — 知道什么是好的代码结构
5. **迭代优化** — 不满意就继续对话，逐步完善

### 常用 AI 编程工具

| 工具 | 类型 | 适合场景 |
|------|------|----------|
| Claude Code | CLI | 命令行项目、复杂重构 |
| Cursor | IDE | 日常开发、代码补全 |
| GitHub Copilot | 插件 | 代码补全、简单问答 |
| Windsurf | IDE | 全栈开发 |

## 💻 一个 Vibecoding 实战例子

**你说：** "帮我写一个 Python 脚本，读取 CSV 文件，统计每列的空值数量，输出到新文件"

**AI 写：** (自动生成完整代码)

**你做：**
1. 快速阅读代码，理解逻辑
2. 运行测试，确认结果正确
3. 如果不对，告诉 AI 哪里需要改
4. 满意后保存到代码片段库

## ⚠️ 容易踩的坑

1. **过度信任 AI** — AI 也会犯错，必须验证
2. **不提问题** — 描述越模糊，产出越差
3. **不学基础** — 完全不理解代码，出了问题无法修复
4. **Copy-Paste 习惯** — 贴了代码就跑，不知道写了什么
