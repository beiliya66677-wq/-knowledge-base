# 📖 高效使用知识库的方法

## 📌 一句话总结
知识库的价值 = 记录频率 × 回顾频率，不是记得多就厉害。

## 🔍 核心理解

### 四个黄金原则

1. **5秒法则** — 有想法时，5秒内打开收件箱记下来，不等「以后」
2. **每周回顾** — 每周末花15分钟整理收件箱，归类到对应文件夹
3. **用自己的话写** — 不要复制粘贴，用自己的理解重写一遍
4. **建立链接** — 用 [[双括号]] 把相关笔记连起来，形成知识网络

### 工作流示意

```
想法闪现 → 0-inbox/ → 每周整理 → 1-learning/ 或 3-code-snippets/
                              ↓
                         补充标签 + 链接
```

### 笔记命名技巧

- ✅ 好名字：`什么是Vibecoding.md`、`Git常用命令.md`、`Python列表推导式.md`
- ❌ 坏名字：`笔记1.md`、`aaa.md`、`临时.md`
- 规则：**一眼就能看出内容**

### 什么时候该记？

| 场景 | 记在哪里 | 例子 |
|------|----------|------|
| 学到一个新概念 | `1-learning/` | 理解闭包、Promise、REST API |
| 调通了一个 bug | `3-code-snippets/` | 错误信息 + 解决方案 |
| 发现一个好工具 | `4-resources/` | 工具名 + 链接 + 为什么好 |
| 有一个新想法 | `0-inbox/` | 临时记录，回头整理 |
| 今天做了什么 | `5-daily/` | 学了什么、遇到什么困难 |

### VS Code 快捷键（打开知识库文件夹后）

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+P` | 快速打开文件 |
| `Ctrl+Shift+F` | 全局搜索所有笔记 |
| `Ctrl+B` | 切换侧边栏 |
| `Ctrl+K V` | 预览 Markdown |

## 💻 代码示例

### 用命令行快速创建笔记

```bash
# 进入知识库
cd ~/Desktop/knowledge-base

# 快速创建一条收件箱笔记
echo "# 新想法" > "0-inbox/$(date +%Y%m%d-%H%M).md"

# 用模板创建学习笔记
cp templates/concept.md "1-learning/vibecoding/新主题.md"

# 查看今天的改动
git diff --stat
```

## ⚠️ 容易踩的坑

1. **完美主义** — 笔记不用写得很漂亮，先记下来，以后再改
2. **只存不看** — 记完就忘 = 白记，每周至少回顾一次
3. **过度分类** — 文件夹不要超过 2 层深，不然找起来麻烦
4. **复制粘贴** — AI 给你的答案，用自己的话重写一遍才能真正理解
